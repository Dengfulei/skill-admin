### 已执行命令

```bash
mvn -v

cd backend && mvn spring-boot:run

cd backend && mvn spring-boot:run -Dspring-boot.run.arguments=--server.port=18080
```

### 通过项

- `mvn test` 通过，后端单测共 `9/9` 成功。
- 本地 MySQL 可连接，`skill_admin` 数据库存在。
- Flyway 迁移成功执行至 `V3`。
- 后端在 `18080` 端口可正常启动，说明 Spring Boot、JPA、Flyway、JWT 依赖链路可运行。
- 登录接口 `/api/auth/login` 返回成功，能正常获取 token。

### 阻塞项

- 默认命令 `mvn spring-boot:run` 使用 `8080` 启动时失败。
- 原因是本机已有 Java 进程占用 `8080` 端口，属于环境占用，不是代码启动失败。

### 风险项

#### 1. 业务异常的 HTTP 状态码语义不一致

- 位置：[GlobalExceptionHandler.java](/Users/dengfulei/Desktop/codex-ai/skill-admin/backend/src/main/java/com/codex/skilladmin/common/GlobalExceptionHandler.java#L14)
- 现状：`BusinessException` 被统一标记为 HTTP `400 BAD_REQUEST`
- 影响：即使业务内部希望表达 `401` 或 `403`，HTTP 响应层仍会固定返回 `400`
- 风险：权限拒绝、未授权等场景的状态码语义可能与验收预期不一致
