# 接口与权限说明

## 认证接口

### `POST /api/auth/login`

登录并返回 JWT。

请求示例：

```json
{
  "username": "admin",
  "password": "123456"
}
```

### `GET /api/auth/me`

返回当前登录用户及部门信息。

## 管理端接口

### `GET /api/admin/resources`

查看当前用户可管理的资源。

- 系统管理员：公共级共享资源
- 部门管理员：本部门部门级资源
- 普通用户：无共享资源管理权限

### `GET /api/user/resources/manageable-personal`

查看当前用户可管理的个人资源。

- 所有登录用户：仅本人个人资源

### `POST /api/admin/resources`

创建资源。

支持：

- `resourceType = SKILL / MCP`
- `scopeLevel = PUBLIC / DEPARTMENT / PERSONAL`
- 指定权限分配 `permissions`

约束：

- 公共级资源仅系统管理员可创建与维护
- 部门级资源仅所属部门管理员可创建与维护
- 个人级资源仅资源本人可创建与维护
- 公共级资源只能配置为全员可用
- 部门级资源只能授权给所属部门，或所属部门内的指定成员
- 个人级资源只能授权给资源本人
- 若 `approvalRequired = true`，则不能预置任何可用权限，只能通过审批生成个人授权
- 已创建资源不得跨层级修改，也不得修改所属部门 / 所属用户

### `PUT /api/admin/resources/{id}`

更新资源及配置。

### `PATCH /api/admin/resources/{id}/enabled`

启用 / 禁用资源。

### `DELETE /api/admin/resources/{id}`

逻辑删除资源。

### `GET /api/admin/applications`

查看当前管理员可审批的申请记录。

- 仅部门管理员可查看本人负责部门的申请
- 系统管理员默认不参与部门级技能审批

### `PATCH /api/admin/applications/{id}/review`

审批申请，审批通过后写入个人使用权限。

## 用户端接口

### `GET /api/user/resources/available`

返回当前用户可访问的资源集合：

- 公共授权
- 部门授权
- 个人授权

### `GET /api/user/resources/apply-catalog`

返回当前用户所在部门、且需要申请、且尚未授权给当前用户的资源。

- 仅普通部门成员可使用该目录
- 系统管理员与部门管理员不走部门技能申请链路

### `GET /api/user/applications`

返回当前用户自己的申请记录。

### `POST /api/user/applications`

提交部门级资源申请。

- 仅普通部门成员可提交
- 系统管理员不通过该链路使用资源
- 部门管理员不通过该链路使用本部门资源，避免出现自申请、自审批

## 运行时鉴权接口

### `POST /api/runtime/invoke`

模拟 AI 在运行时发起 Skill / MCP 调用。

请求示例：

```json
{
  "resourceCode": "sales-analytics-mcp"
}
```

鉴权逻辑：

1. 资源必须存在
2. 资源必须启用且状态为 `ACTIVE`
3. 按资源归属范围执行校验
   - 公共级资源：仅 `PUBLIC + USE` 生效
   - 部门级资源：当前用户必须仍属于资源所属部门，再命中 `DEPARTMENT + USE` 或审批生成的 `PERSONAL + USE`
   - 个人级资源：当前用户必须是资源本人，且命中 `PERSONAL + USE`

失败时返回友好提示：

> 当前账号暂无该技能或工具的使用权限，可联系管理员开通或提交部门技能申请。

## 服务端权限边界

### 公共级资源

- 仅系统管理员可维护
- 全员可用

### 部门级资源

- 仅该部门管理员可维护
- 可按所属部门直接授权
- 也可开启申请模式，由管理员审批后按个人授权
- 开启申请模式后，资源不会再保留或新增部门直授权限
- 即使存在个人授权，用户离开所属部门后也不能继续使用

### 个人级资源

- 仅本人可维护
- 仅本人可用

## 当前前端入口规则

- 所有登录用户均可访问 `/user/manage-resources`，仅管理自己的个人资源
- 仅普通部门成员显示 `/user/applications` 入口
- 系统管理员仅可访问公共级共享资源管理
- 部门管理员可访问本部门共享资源管理与本部门申请审批
