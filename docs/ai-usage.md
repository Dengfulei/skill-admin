# AI 辅助编程（Codex）应用说明

## 文档定位

本文档用于说明本项目如何在完整开发过程中使用 Codex 进行 AI 辅助编程，并重点回答评审通常会关注的 4 个问题：

- AI 介入了哪些环节
- AI 具体产出了什么
- 人工做了哪些判断与修正
- AI 是否实质性提升了开发效率与交付完整度

## AI 使用目标

本项目全程使用 Codex 辅助以下任务：

- 需求分析与模块拆分
- 数据库 Schema 设计
- Spring Boot 3 后端骨架搭建
- Vue3 + Element Plus 前端页面生成与样式调整
- 权限模型与运行时鉴权逻辑实现
- 交付文档、演示脚本与过程材料整理

## AI 使用原则

项目在使用 AI 生成代码时遵循以下原则：

1. AI 负责提速，不替代关键业务判断
2. 权限边界、数据库设计和运行时鉴权由人工复核
3. AI 生成的 SQL、接口和页面代码必须经过实际运行验证
4. 重要设计决策需要形成文档，而不是只停留在生成结果

## AI 介入阶段概览

### 1. 需求拆解阶段

输入目标：

- 分析赛题要求
- 提炼显性任务与隐性考点
- 拆分为数据库、权限模型、管理台、用户端、运行时鉴权、交付材料 6 个模块

AI 产出：

- 统一资源模型思路
- 三级权限边界定义
- 推荐实施顺序

人工修正：

- 明确后端技术栈为 `Spring Boot 3 + MySQL 8.0`
- 明确前端技术栈为 `Vue3 + Element Plus`
- 将“维护权限”和“使用权限”区分处理

### 2. Schema 设计阶段

输入目标：

- 将 Skill / MCP 文件配置迁移为数据库结构
- 支撑公共 / 部门 / 个人三级权限
- 支撑部门技能申请审批流程

AI 产出：

- 主表 `skill_resource`
- 详情表 `skill_config`、`mcp_config`
- 权限表 `resource_permission`
- 申请表 `resource_apply`

人工修正：

- 采用 MySQL 8.0 `JSON` 字段承接原始配置
- 增加 `approval_required` 支持“需申请后使用”
- 通过 Flyway 管理建表和初始化数据

### 3. 后端实现阶段

输入目标：

- 搭建 Spring Boot 3 工程
- 实现登录、资源管理、申请审批、运行时鉴权接口

AI 产出：

- Maven 工程骨架
- Flyway 初始化 SQL
- 统一返回体与异常处理
- 资源、申请、鉴权相关 Controller / Service / Entity / Repository

人工修正：

- 调整 JWT 中部门集合的解析方式
- 修正默认启用状态与审批开关的安全默认值
- 使用真实 MySQL 账号启动后端并验证初始化成功

### 4. 前端实现阶段

输入目标：

- 生成 Vue3 + Element Plus 管理台与用户端页面
- 覆盖登录、资源管理、申请审批、我的可用资源、部门技能申请等主流程

AI 产出：

- 登录页
- 管理端资源列表页
- 管理端申请审批页
- 用户端资源列表页
- 用户端申请页

人工修正：

- 调整页面结构以适应答辩演示路径
- 增加“运行鉴权”按钮便于展示后端权限校验
- 修复头部高度导致的标题遮挡问题
- 统一字体、间距、容器层级与表格可读性

### 5. 交付整理阶段

输入目标：

- 输出 README、Schema 文档、接口说明、演示脚本和 AI 使用说明

AI 产出：

- 文档初稿
- 演示流程描述
- 交付检查思路

人工修正：

- 对照比赛评分项补齐设计决策说明
- 强化过程记录材料，避免只剩“结果文档”

## AI 对开发效率的实际价值

在本项目中，Codex 的价值主要体现在 5 个方面：

1. 快速完成需求结构化分析，减少前期反复试错
2. 快速生成数据库模型和接口骨架，缩短从 0 到 1 的搭建时间
3. 自动生成高重复度 CRUD 与表单页面，节省大量样板代码时间
4. 帮助同步沉淀文档与演示材料，减少后期补文档成本
5. 能在联调阶段持续辅助定位问题并快速迭代修复

## 人工负责的关键决策

以下内容由人工重点把关，未完全交由 AI 决定：

- 技术栈选型
- 权限边界定义
- MySQL 表设计与字段语义
- 运行时鉴权规则
- 演示流程与交付清单对齐

## 项目中的 AI 使用结论

Codex 在本项目中并非只用于“写代码”，而是贯穿了需求拆解、建模设计、实现落地、联调修复和交付整理全过程。AI 明显提升了首版搭建速度和文档沉淀效率，但最终系统的权限正确性、交付完整性和答辩表达仍由人工负责校验与收口。
