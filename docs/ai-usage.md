# AI 辅助编程（Codex）应用说明

## 使用目标

本项目全程使用 AI 辅助编程工具 Codex 参与设计与实现，目标是提升以下阶段的交付效率：

- 需求拆解
- 数据库 Schema 设计
- Spring Boot 后端骨架搭建
- Vue3 前端页面生成
- 演示脚本和文档整理

## 使用过程记录

### 1. 需求拆解阶段

输入目标：

- 识别题目中的显性功能要求与隐性考点
- 拆分为数据库、权限模型、管理台、用户端、运行时鉴权、文档六个模块

AI 产出：

- 统一资源模型思路
- 三级权限的边界说明
- 推荐的实施顺序

人工修正：

- 明确后端技术栈为 `Spring Boot 3 + MySQL`
- 明确前端技术栈为 `Vue3 + Element Plus`

### 2. Schema 设计阶段

输入目标：

- 将 Skill / MCP 文件配置迁移为数据库结构
- 支撑公共 / 部门 / 个人三级权限和部门技能申请

AI 产出：

- 主表 `skill_resource`
- 详情表 `skill_config` 与 `mcp_config`
- 权限表 `resource_permission`
- 申请表 `resource_apply`

人工修正：

- 选择 MySQL 8.0 `JSON` 字段承接原始配置
- 增加 `approval_required` 支撑部门级申请模式

### 3. 后端实现阶段

输入目标：

- 搭建 Spring Boot 3 工程
- 实现登录、资源管理、申请审批、运行时鉴权接口

AI 产出：

- `pom.xml`
- Flyway 初始化 SQL
- 统一返回结构与异常处理
- 资源、申请、鉴权控制器和服务代码

人工修正：

- 调整 JWT 中部门集合的解析方式
- 将默认启用和审批开关补成安全默认值

### 4. 前端实现阶段

输入目标：

- 生成 Vue3 + Element Plus 管理台与用户端页面

AI 产出：

- 登录页
- 资源管理页
- 申请审批页
- 我的可用技能页
- 部门技能申请页

人工修正：

- 将页面组织成更适合答辩演示的导航结构
- 增加“模拟调用鉴权”按钮，便于展示运行时权限校验

### 5. 交付整理阶段

输入目标：

- 输出 README、Schema 文档、演示脚本和 AI 使用说明

AI 产出：

- 可直接打包进交付物的文档初稿

人工修正：

- 按比赛要求补充“评分点对应关系”和演示顺序

## AI 的实际价值

- 将需求拆解速度明显提升
- 快速生成数据库与接口骨架
- 减少重复性 CRUD 和页面模板编码
- 加速文档沉淀，使技术决策可解释、可答辩

## 人工负责的关键部分

- 最终技术栈与边界控制
- 权限语义校对
- 运行时鉴权规则确认
- 交付内容与比赛评分点对齐
