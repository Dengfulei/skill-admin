# 智能辅助编程说明及过程记录

## 文档定位

本文档整合《智能辅助编程说明》和《智能辅助编程过程记录台账》，用于统一说明本项目在开发全过程中如何使用 Codex 进行 AI 辅助编程，并保留关键过程证据。文档重点回答以下问题：

- AI 介入了哪些环节
- AI 具体产出了什么
- 人工做了哪些判断与修正
- AI 是否实质性提升了开发效率与交付完整度

本文档兼顾“方法说明”和“过程留痕”，可直接作为项目 AI 辅助编程应用说明与过程记录材料。

## 项目信息

- 题目：V03｜Skill/MCP 配置数据库化与权限管控
- 技术栈：Spring Boot 3、MySQL 8.0、Vue3、Element Plus
- 工作目录：`/Users/dengfulei/Desktop/codex-ai/skill-admin`
- 主要 AI 工具：Codex

## AI 使用目标

本项目全程使用 Codex 辅助以下任务：

- 需求分析与模块拆分
- 数据库 Schema 设计
- Spring Boot 3 后端骨架搭建
- Vue3 + Element Plus 前端页面生成与样式调整
- 权限模型与运行时鉴权逻辑实现
- 交付文档、演示脚本与过程材料整理

## AI 使用原则

项目在使用 AI 生成代码时遵循以下原则：

1. AI 负责提速，不替代关键业务判断
2. 权限边界、数据库设计和运行时鉴权由人工复核
3. AI 生成的 SQL、接口和页面代码必须经过实际运行验证
4. 重要设计决策需要形成文档，而不是只停留在生成结果

## AI 介入阶段概览

### 1. 需求拆解阶段

AI 主要参与赛题分析、显性任务与隐性考点提炼，并将项目拆分为数据库、权限模型、管理台、用户端、运行时鉴权、交付材料 6 个模块。

AI 产出：

- 统一资源模型思路
- 三级权限边界定义
- 推荐实施顺序

人工修正：

- 明确后端技术栈为 `Spring Boot 3 + MySQL 8.0`
- 明确前端技术栈为 `Vue3 + Element Plus`
- 将“维护权限”和“使用权限”区分处理

### 2. Schema 设计阶段

AI 参与将 Skill / MCP 文件配置迁移为数据库结构，并围绕公共、部门、个人三级权限及审批流进行建模。

AI 产出：

- 主表 `skill_resource`
- 详情表 `skill_config`、`mcp_config`
- 权限表 `resource_permission`
- 申请表 `resource_apply`

人工修正：

- 采用 MySQL 8.0 `JSON` 字段承接原始配置
- 增加 `approval_required` 支持“需申请后使用”
- 通过 Flyway 管理建表和初始化数据

### 3. 后端实现阶段

AI 参与 Spring Boot 3 工程搭建，以及登录、资源管理、申请审批、运行时鉴权链路实现。

AI 产出：

- Maven 工程骨架
- Flyway 初始化 SQL
- 统一返回体与异常处理
- 资源、申请、鉴权相关 Controller / Service / Entity / Repository

人工修正：

- 调整 JWT 中部门集合的解析方式
- 修正默认启用状态与审批开关的安全默认值
- 使用真实 MySQL 账号启动后端并验证初始化成功

### 4. 前端实现阶段

AI 参与 Vue3 + Element Plus 管理台与用户端页面生成，并辅助页面样式优化与演示路径整理。

AI 产出：

- 登录页
- 管理端资源列表页
- 管理端申请审批页
- 用户端资源列表页
- 用户端申请页

人工修正：

- 调整页面结构以适应答辩演示路径
- 增加“运行鉴权”按钮便于展示后端权限校验
- 修复头部高度导致的标题遮挡问题
- 统一字体、间距、容器层级与表格可读性

### 5. 交付整理阶段

AI 参与 README、Schema 文档、接口说明、演示脚本与 AI 使用材料整理。

AI 产出：

- 文档初稿
- 演示流程描述
- 交付检查思路

人工修正：

- 对照比赛评分项补齐设计决策说明
- 强化过程记录材料，避免只剩“结果文档”

## AI 价值与人工校验边界

### AI 对开发效率的实际价值

在本项目中，Codex 的价值主要体现在 5 个方面：

1. 快速完成需求结构化分析，减少前期反复试错
2. 快速生成数据库模型和接口骨架，缩短从 0 到 1 的搭建时间
3. 自动生成高重复度 CRUD 与表单页面，节省大量样板代码时间
4. 帮助同步沉淀文档与演示材料，减少后期补文档成本
5. 能在联调阶段持续辅助定位问题并快速迭代修复

### 可由 AI 快速生成初稿的内容

- SQL 草案
- CRUD 页面
- 接口骨架
- 文档初稿
- 演示步骤梳理

### 必须人工校验的内容

- 权限语义是否正确
- 数据模型是否可扩展
- 运行时鉴权是否真正落在服务端
- 页面是否符合实际演示场景
- 交付物是否满足比赛要求

### 人工负责的关键决策

- 技术栈选型
- 权限边界定义
- MySQL 表设计与字段语义
- 运行时鉴权规则
- 演示流程与交付清单对齐

## 过程结论

本项目中的 AI 使用并非停留在“生成几段代码”，而是形成了完整闭环：

1. AI 参与需求拆解和技术路线确定
2. AI 参与数据库建模和系统骨架生成
3. AI 参与前后端实现与联调修复
4. AI 参与文档与交付材料整理
5. 人工对业务边界、权限正确性和交付完整性进行最终把关

Codex 在本项目中并非只用于“写代码”，而是贯穿了需求拆解、建模设计、实现落地、联调修复和交付整理全过程。AI 明显提升了首版搭建速度和文档沉淀效率，但最终系统的权限正确性、交付完整性和答辩表达仍由人工负责校验与收口。
