# 演示脚本

建议将演示控制在 5 分钟内，按下面顺序执行。

## 1. 管理员创建公共技能

- 使用 `admin / 123456` 登录
- 打开资源管理页
- 创建一个 `PUBLIC` 范围的 Skill
- 默认分配为全员可用

要点说明：

- 公共级资源只有系统管理员可维护
- 创建后所有用户都能在“我的可用技能”中看到

## 2. 部门管理员创建部门级 MCP

- 切换到 `sales_admin / 123456`
- 新建 `DEPARTMENT` 范围的 MCP
- 所属部门选择销售部
- 打开“需要申请”开关

要点说明：

- 部门资源由部门管理员维护
- 本例设置为“审批后使用”，用于演示申请流

## 3. 普通员工查看资源差异

- 使用 `sales_user / 123456` 登录
- 进入“我的可用技能”
- 展示能看到公共资源，但未自动获得需要申请的销售 MCP
- 进入“部门技能申请”，看到销售 MCP 出现在可申请目录

- 使用 `hr_user / 123456` 登录
- 展示只能看到公共资源，看不到销售部 MCP

## 4. 申请与审批

- `sales_user` 提交销售 MCP 申请
- 切换回 `sales_admin`
- 在“申请审批”页通过该申请

要点说明：

- 审批通过后，系统会增加一条 `PERSONAL + USE` 权限记录

## 5. 运行时鉴权

- 回到 `sales_user`
- 在“我的可用技能”页点击“运行鉴权”
- 展示调用通过

- 再使用 `hr_user`
- 在“我的可用技能”页的“运行时鉴权测试”输入框中输入 `sales-analytics-mcp`
- 点击“验证权限”
- 返回友好提示：
  - 当前账号暂无该技能或工具的使用权限，可联系管理员开通或提交部门技能申请

建议在录屏时补一句说明：

- 当前页面上普通“运行鉴权”按钮只会出现在“当前用户可用资源”列表中，所以天然多为成功场景
- 为了验证“无权限返回友好提示”，这里通过手动输入资源编码触发运行时鉴权
- `hr_user` 属于人事部，不具备销售部 `sales-analytics-mcp` 的使用权限，因此后端会明确返回拒绝提示

也可以补一个申请前对照演示：

- 使用 `sales_user`
- 在“部门技能申请”页找到 `销售分析 MCP`
- 点击“验证无权限”
- 申请前同样会返回无权限友好提示
- 审批通过后，再回到“我的可用技能”页输入同一编码验证，会变成“鉴权通过，可执行 Skill/MCP 调用”

## 6. 个人级资源

- 使用 `alice / 123456`
- 展示 Alice 能看到自己的个人资源
- 其他账号不可见

## 7. 收尾总结

- 配置已从文件迁移为数据库统一管理
- 实现公共 / 部门 / 个人三级权限
- 管理台、用户端、运行时鉴权都已打通
